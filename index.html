Timing and error checking the code

With all projects or even updates we consider error testing and unit testing to be the staple of any additions to a code base. If unit testing is not part of the framework build we would usally run something like the following in PHP:

$start = microtime(true);
...(any code)
$time_elapsed = microtime(true) - $start;
error_log('run time = '.$time_elapsed);

Using the above method within a PHP project can be extremely useful in identifying bottlenecks in the code and areas that need further optimisation. If we suspect their maybe a wait event on the front end within JavaScript we would inject the following into the JS code:

var start = new Date().getTime();
...(any code)
var end = new Date().getTime();
console.log('this many milliseconds passed', end - start);

Of course its not just slow code or functions that can be an issue sometimes the code is just plain broke, this is why console logs and error logs are so important. In Javascript errors and warnings are usally passed to the console log where the issue can be seen, however sometimes there is is no visible error because the value passed could just be not as expected, thats when its useful to add console.log(a value) to the script. A simular practice can be used with PHP in adding error_log(a value) to a script and check the default error log for the results.
